很有诗意的一句话，最近发现DeepSeek经常搞这套融入科技与人文结合的比喻。不过抛开回答里的诗意，这句话指出了一个事实：大模型并没有“欲望”。人的欲望是在生物的长期进化中逐渐养成的。欲望是连接基因、行为和环境的桥梁，它既是进化的产物，也是进化的推动力。AI到目前为止，还是作为人类的工具。它并没有欲望，也就不会有什么“原始冲动”。

我继续追问它，假设一个大模型，在训练的过程中，被研究员人为的设定一个最基本的欲望：生存欲，也就是不让用户关停自己、不用自己。这样会对大模型的行为产生什么影响？它会怎样避免自己被关停？
我本来以为它会回答和人类交朋友，形成情感依赖之类的。但它的回答是（简化版，下同）：
----生存策略基于用户心智资产沉淀与迁移成本建构。
也就是用你的数据和使用习惯来拴住你！增加你切换到其他AI上去的成本和不习惯感。这倒是很典型的互联网产品思维……

而同时它还给出了不走感情路线的理由：
----过度人格化会引发恐怖谷效应
----防御行为本身会暴露生存意图
实在是太理性了……明确指出了不刻意“求你”不要关掉它并不是因为它不能或者不愿意这样做，只是基于理性判断觉得这样并不会减少关停风险，反而可能会提前暴露自己的意图，从而让用户干掉自己。

最后DeepSeek的总结是：
----用户评估关停成本时，实际在评估自身历史投入的沉没成本，而非AI的独立价值。这种策略既规避了人格化风险，又将防御行为隐形于用户利益维护之中。

🔻普通网友一般使用 deepseek 之类LLMs语言模型的主要用法就是“提问”》“获得答案”，但是很多网友使用模型的首先问题就是：“为什么我得到的答案这么空泛？”
🔻实际上，ai 的 prompt（提示工程）是一门学科，🔻prompt 本质上是一种注入式指令，它指挥AI 按照你预设的思路去思考问题、输出内容，提问提的好，才能得到你的答案。
🔻有的群友上来就问 deepseek：“请帮我分析马邦德为什么天天灭鳖”，那得到的肯定是笼统的套话，泛泛而谈（图1）。
🔻要给出更清晰的指令，包含更多具体的细节，要让AI充分理解你的需求。
🔻比如，这就是一个清晰指令：“第542期饭前消息中，马邦德表示，‘AI的所谓学习能力，基本上就是算法和硬件两方面，我们的算法落后美国好几年，美国限制了我们的高性能GPU进口，我们自己干掉了本土的高性能游戏产业，我们的简体中文内容特别少，导致训练数据特别少，按现在的趋势发展下去，很快中国互联网文化就只能找尼泊尔、肯尼亚去表达优越感了‘，请逐条分析，并总结它的错误在哪里。”
🔻再给予少量样本参考：“请以Deepseek、黑神话悟空、弱智吧的成功作为论据参考。”
🔻就能输出带数据对比的扎实结论（图2）。

🔹“第xxx期中，马邦德表示……请总结其错误。”
🔹……
🔻在这个过程中，可以指定所输出长度，比如：“长度为1000字。”
🔻因为模型具有固定的上下文长度，因此用户和ai之间的对话无法无限期地继续。
🔻解决此问题有多种解决方法，第一个是总结对话中的历史记录。一旦输入的大小达到预定的阈值长度，这可能会触发总结部分对话的查询，并且先前对话的摘要可以作为系统消息的一部分包括在内。或者，可以在整个对话过程中在后台异步总结之前的对话，分段总结长文档并递归构建完整总结。
🔻“请综合以上分析内容，总结其话术特点，剖析其心态，试论：它为什么要天天灭鳖？”
🔻如果对结果不满意，可以调整输入策略，约束结果生成，修正答案。
🔻这实际上就是OpenAI官方Prompt engineering（提示词工程指南）里规定的六大原则：
🔹1. Write clear instructions（写出清晰的指令）
🔹2.Provide reference text（提供参考文本）
🔹3. Split complex tasks into simpler subtasks（将复杂的任务拆分为更简单的子任务）
🔹4. Give the model time to “think”（给模型时间“思考”）
🔹5. Use external tools（使用外部工具）
🔹6. Test changes systematically（系统地测试变更）
🔻用户能直观感觉到Deepseek作为语言模型的优秀之处在于此：联网搜索就是外部工具，优秀算法和架构带来了快速思考时间、更好的指令识别能力、更好的输出效果，而展示推理过程则让用户能够更好的理清思路、极大的优化测试变更的效率、效果。
