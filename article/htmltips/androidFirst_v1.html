2. 安装 JDK

  • 配置 JAVA_HOME、CLASSPATH 和 PATH 环境变量是使用 Java 开发的基础步骤。以下是在 Windows 系统上配置这些环境变量的详细步骤：
  • 1. 找到 JDK 的安装路径
  • 首先，你需要找到 JDK 的安装路径。通常情况下，它会被安装在 C:\Program Files\Java 目录下。在这个目录下，你会看到类似 jdk-17.0.2 或 jdk-1.8.0_361 这样的文件夹，这就是 JDK 的安装目录。记下这个路径，稍后会用到。
  • 2. 打开系统属性
  • Windows 10/11: 在任务栏的搜索框中输入“环境变量”，然后选择“编辑系统环境变量”。或者，你也可以通过“控制面板” -> “系统和安全” -> “系统” -> “高级系统设置”来打开系统属性。
  • 3. 点击“环境变量”按钮
  • 在“系统属性”窗口中，点击“环境变量”按钮。
  • 4. 配置 JAVA_HOME 变量
  • 在“系统变量”区域，点击“新建”按钮。在“变量名”中输入 JAVA_HOME。在“变量值”中输入你的 JDK 安装路径，例如 C:\Program Files\Java\jdk-17.0.2（根据你的实际安装路径修改）。点击“确定”。
  • 5. 配置 CLASSPATH 变量 (可选，JDK 9 及以上版本通常不需要)
  • 在“系统变量”区域，点击“新建”按钮。在“变量名”中输入 CLASSPATH。在“变量值”中输入 .;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar（注意最前面的 . 代表当前目录）。点击“确定”。注意： 从 JDK 9 开始，CLASSPATH 环境变量通常不再需要手动设置。Java 启动器会自动搜索类路径。但是，某些旧的应用程序或框架可能仍然需要它。
  • 6. 配置 PATH 变量
  • 在“系统变量”区域，找到名为 Path 的变量，选中它，然后点击“编辑”按钮。在弹出的“编辑环境变量”窗口中，点击“新建”按钮。输入 %JAVA_HOME%\bin。点击“确定”。注意： 确保 %JAVA_HOME%\bin 在 Path 变量的值中靠前的位置，这样系统会优先使用你安装的 JDK 中的工具。
  • 7. 确认配置
  • 点击所有打开的窗口上的“确定”按钮，保存更改。
  • 8. 验证配置
  • 打开命令提示符（或 PowerShell）。输入 java -version 并按回车键。如果正确配置了环境变量，你将看到 JDK 的版本信息。输入 javac -version 并按回车键。如果正确配置了环境变量，你将看到 Java 编译器的版本信息。
  • 示例：假设你的 JDK 安装在 C:\Program Files\Java\jdk-17.0.2 目录下，那么配置后的环境变量应该是： JAVA_HOME: C:\Program Files\Java\jdk-17.0.2 CLASSPATH: .;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar (可选) Path: 在原有的基础上添加 %JAVA_HOME%\bin
  • 注意事项：确保路径的正确性，包括大小写和空格。如果修改了环境变量后命令提示符窗口已经打开，需要关闭并重新打开一个新的命令提示符窗口，才能使更改生效。如果安装了多个 JDK 版本，JAVA_HOME 变量将决定使用哪个版本。
